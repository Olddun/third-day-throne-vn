本文档沉淀《第三十日王座》视觉小说开发过程中用户提出的全部核心要求、体验问题、修正经验和后续验收红线。它不是阶段汇报，而是后续继续制作时必须遵守的产品准则。

## 一句话目标

做一部遵循“救猫咪”剧作节拍、面向年轻宅男玩家、从第一分钟到终局都能持续驱动玩家点击下去的商业级视觉小说。

最终目标不是“能播放剧情”，而是：

- 总游玩时长约 20 小时。
- 10 个以上有实质意义的关键分歧点。
- 15 个以上结局。
- 80 张以上 CG。
- 每个主要人物都有自己的欲望、伤口、面具和弧光。
- 整体剧情脉络必须在中后段大幅偏离玩家开局预期。
- 核心点必须是好玩。不好玩禁止验收。

## 用户明确提出过的体验要求

| 领域 | 必须做到 | 反例/踩坑 |
| --- | --- | --- |
| 开场钩子 | 第一时间给玩家危险、谜团、身份和选择压力。 | 开头只是介绍人物、送上一只猫，会显得没劲。 |
| 玩家身份 | 玩家必须清楚自己在扮演谁、以什么立场做选择。 | 只显示选项，不说明玩家是塞德里克，会让选择失去代入。 |
| 选项设计 | 两个或多个选项都要看似有道理，短但有行动感，并且有收益和代价。 | 选项太简单、显然正确、没有刺激感。 |
| 文本节奏 | 每句台词短一点，拆成多句；角色说话文本量要足够，有表情。 | 长段解释背景，玩家没有点击动力。 |
| 人物引入 | 开局不要一口气堆详细人物设定，要通过行动逐步认识角色。 | 一上来铺人物履历，读者不方便进入。 |
| 立绘系统 | 每个人说话时必须有对应立绘；场上角色显示，当前发言者正常，其他在场者暗淡；没说话且不在场者隐藏。 | 开场只有猫立绘，会让人误以为只有猫在说话。 |
| 旁白显示 | 旁白不需要身份标识；具名角色要有身份标识。 | 旁白显示“旁白”会显得廉价。 |
| 猫立绘位置 | 猫不能被对话框遮住，位置要优雅。 | 其他角色不遮，只有猫被遮，视觉上很明显。 |
| 表情差分 | 立绘需要多表情差分，发言时能体现情绪。 | 单表情立绘无法支撑长篇情绪变化。 |
| CG | 关键人物出场要有 CG；每章/每个小故事至少 3 张 CG。 | 只有立绘推进会缺少商业 VN 的仪式感。 |
| 动画 CG | 关键人物出场的动画 CG 要有动画感觉，人物肢体和表情要符合剧情。 | 只做轻微晃动或重影效果不够。 |
| 动画质量 | 去除所有可能带来重影的动画效果，效果出现后必须自检。 | 叠多层局部图造成残影。 |
| 音乐 | 音乐音调不能单一，使用无版权古典乐。 | 单调循环容易疲劳。 |
| VN 操作 | 对话框下方要有常见 VN 操作：历史、自动、已读快进、分支、快存、快读、设置。 | 玩家找不到 VN 标准功能会降低完成度。 |
| 历史记录 | 滚轮向上滚动可以看历史。 | 只能点按钮不够符合习惯。 |
| 快捷键 | 需要快速存档/读档快捷键。 | 没有 F5/F9 一类快速操作，VN 手感不足。 |
| 分支树 | 需要剧情分支树状图或路线回顾。 | 只把分支罗列出来、没有上下文，会显得没价值。 |
| 托管交付 | 做完后要推到云端网页，给可玩网址，并通过飞书发送。 | 只本地可跑不算交付。 |

## 后续长篇版的体验模块库

这些体验模块可以像卡牌一样塞入不同章节，但每次使用都必须服务玩家当下目标。

| 模块 | 用途 | 适用位置 |
| --- | --- | --- |
| 倒计时处决 | 让玩家立刻面对会死人、会丢证据的压力。 | 序章第一幕 |
| 敌国公主伪装成猫 | 用可爱外壳包住危险、尊严和复仇。 | 序章至第一章 |
| 坏结局给情报 | 失败不是惩罚，而是下一周目的线索奖励。 | 全篇 |
| 密信触发器 | 用王印、银铃、梦境、密信制造选择与延迟后果。 | 第一章后 |
| 闭锁宫廷疑云 | 把朝堂做成封闭村落式猜疑场。 | 第一至第四章 |
| 玩家身份翻转 | 中盘揭示塞德里克也被切割过记忆。 | 中点反转 |
| 糖里藏刀 | 亲密场景背后同时有政治或记忆代价。 | B 故事线 |
| 谎言地图 | 分支树不只回顾选择，还回顾已知谎言。 | 第二章解锁 |
| 信任与真相互斥 | 追查真相会伤害她，只安抚她会错过证据。 | 关键选择 |
| 王座面具与私伤 | 强势身份和私密脆弱同时满足代入。 | 全篇 |
| 周目信息继承 | 坏结局后保留梦痕、密语、CG 注释。 | 第二轮后 |
| 恋爱也是策略 | 亲近改变盟友、审判和牺牲顺序。 | 全篇 |
| 盟友各有目的 | 所有配角都会在玩家犹豫时主动行动。 | 全篇 |
| 中点假胜利 | 洗清战争真相后发现真相本身是诱饵。 | 第六章 |
| 最低谷亲密 | 失去王位和人形，只剩彼此真实记忆。 | 第九章 |
| 路线锁真相 | 不同结局开放不同幕后证据。 | 结局群 |
| CG 承担戏剧瞬间 | 出场、亲密、坏结局、反转都用 CG 兑现。 | 全篇 |
| 终局重解释开场 | 最后让开场处决令、王印、金丝笼变成同一仪式的碎片。 | 真结局 |

## 主要人物弧光原则

| 人物 | 欲望 | 伤口 | 面具 | 弧光 |
| --- | --- | --- | --- | --- |
| 塞德里克 | 坐稳王座，查清先王留下的战争伪证。 | 十六岁被迫学会用冷酷保命，并被切走关键记忆。 | 把所有善意伪装成利用。 | 从把人当棋子，到愿意让真相审判自己。 |
| 安塔莉亚 | 夺回人形，救下族人，找出灭国真凶。 | 亲眼看见王都焚毁，却不确定自己是否也参与仪式。 | 用猫形沉默和尖锐敌意保护自尊。 | 从只想复仇，到选择把故国未来交给活人共同承担。 |
| 大将军 | 维持军权，阻止战争罪责反噬北境军。 | 曾在旧王都火海中救过孩子，却背上屠城名声。 | 用粗暴忠诚掩盖恐惧。 | 从要求血债血偿，到承认军队也需要被审判。 |
| 财政大臣 | 保住贵族秩序与账册秘密。 | 少年时因饥荒失去家族，只相信账本和利益交换。 | 把所有善恶都说成国家成本。 | 可成为揭账盟友，也可成为结局线的刽子手。 |
| 老书记官 | 让先王时代被抹掉的名字重新出现。 | 参与过第一次记忆切割仪式，余生都在补偿。 | 用温和旁观掩盖知情不报。 | 从记录者变成证人，用自己记忆补齐真相。 |
| 侍女米拉 | 在宫廷夹缝中活下去，并保护小人物。 | 家人死于王都粮道断供。 | 绝对服从。 | 从递盘子的旁观者，到打开宫门的关键行动者。 |
| 侍从卢卡 | 摆脱底层身份，成为被王看见的人。 | 曾告发同伴换取入宫机会。 | 卑微勤快。 | 可被玩家拉回，也可因嫉妒投向政变线。 |

## 救猫咪节拍如何落地

| 节拍 | 本作对应 | 玩家当下目的 |
| --- | --- | --- |
| Opening Image | 处决倒计时、金丝笼、王印证物同场出现。 | 判断自己要当怎样的国王。 |
| Theme Stated | 真相死得太快，活人才有机会作证。 | 理解杀伐果断与追真相不能总兼得。 |
| Setup | 宫廷、战俘营、猫形安塔莉亚、三项核心数值建立。 | 每五分钟获得新疑点或关系反馈。 |
| Catalyst | 密信证明王印来自未来周目。 | 意识到自己可能已经历过这三十日。 |
| Debate | 查账、安抚、诱敌三线互斥。 | 为玩法风格付代价。 |
| Break into Two | 安塔莉亚月下现形。 | 从保护转向共犯关系。 |
| B Story | 恋爱线影响政治线和推理线。 | 亲密场景也获得情报、危险和情绪回报。 |
| Fun and Games | 宫廷疑云、营地调查、旧都潜入、梦境审讯轮替。 | 持续解谜、收集坏结局线索、解锁 CG。 |
| Midpoint | 洗清安塔莉亚后发现塞德里克是仪式容器。 | 过去预期被推翻。 |
| Bad Guys Close In | 将军逼宫、大臣烧账、侍从背叛、书记官失忆。 | 决定优先救谁、牺牲什么。 |
| All Is Lost | 王位失去，安塔莉亚人形崩解。 | 用此前积累换最后一次机会。 |
| Finale | 审判、破咒、献祭、共治、放逐等结局群展开。 | 让自己的选择成为结局。 |

## 分支设计红线

- 选项按钮只写行动，不写理由和数值。
- 每组选项都要有诱惑和代价，不能出现显然正确的按钮。
- 选项后下一段必须立刻兑现：关系变化、政治压力、情报增益、坏结局伏笔至少出现一个。
- 亲近、警戒、观察不能只是内部数值；玩家要能从表情、台词、分支回顾里感到变化。
- 分支树不能只是“你选了什么”，还要告诉玩家当时的语境和被改变的局势。
- 坏结局必须给新情报、新 CG 或新路线暗号，否则删掉。

## 美术和音频经验

- 每个关键人物出场要有 CG。
- 每章或每个小故事至少 3 张 CG。
- 长篇目标为 80 张以上 CG，当前规划 84 张。
- 动画 CG 要像人物真的在动：肢体、表情和剧情动作一致。
- 避免多层人物叠图造成重影；CG 动画优先移动单张图，叠光效时不能复制人物主体。
- 所有角色至少 3 个表情差分，重要角色情绪推进要继续扩展差分。
- 猫体型特殊，必须单独检查是否被对话框遮挡。
- 音乐使用无版权古典乐，并按场景切换情绪，避免单调循环。

## UI 和系统经验

- 对话框底部必须有 VN 常见操作：历史、自动、快进、分支、快存、快读、设置。
- 上滚打开历史记录。
- 快捷键快速存档/读档。
- 已读文本快进要尊重已读状态，不能跳过新文本。
- 旁白隐藏身份标识，具名角色显示身份标识。
- 场上所有在场者显示，发言者高亮，其余暗淡，不在场者隐藏。
- 分支图要做成路线回顾，后续长篇版要升级成“谎言地图”和“结局情报图谱”。

## 工程验收经验

- 用户指出体验问题后，不能只修眼前症状，要把同类问题写成测试或规则。
- 前端/游戏 UI 修改后，要真实运行验证桌面和移动布局。
- 纯单元测试无法覆盖视觉问题时，要补半集成或浏览器自玩证据。
- 完成后要能回答三件事：解决了什么问题，消费者是谁，消费者是否已经能正常用上。
- 对已经不满意的体验问题，功能必须有可见兑现感，不能做“看起来有但没反馈”的按钮。
- 长篇目标没有完成前，不能因为短篇原型可玩就标记 goal 完成。

## 当前状态和下一步

当前网页版本仍是第一阶段可玩原型，不是 20 小时完整成品。已经完成的关键推进是：

- 开场从“人物介绍”改成“刑台处决、王印证物、金丝笼同时压上来”。
- 前两个选择有明确玩家身份、收益和代价。
- 长篇版蓝图已经进入代码，并由测试验证数量和字段。
- “不好玩禁止验收”已经写入 `funCheckpoints` 和 `test/opening-hooks.mjs`。

下一阶段应该优先做：

1. 把 `web/story-plan.js` 的蓝图拆成真正可玩的长篇章节数据。
2. 建立 15 个结局的路由、结局锁和坏结局情报继承。
3. 生成并接入第一批新增 CG，优先覆盖开场、月下现形、中点反转、坏结局。
4. 把分支回顾升级为谎言地图，记录玩家已知真相、误导信息和结局情报。
5. 每扩一章都跑“好玩性检查”：即时目标、阻力、奖励、下一钩子、选项代价全部存在后才验收。
